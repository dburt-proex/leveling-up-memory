# Sessions

This folder stores chronological session logs for the Leveling Up operating system.

## Purpose
- preserve session continuity
- track what was built
- capture project movement
- record next actions
- prevent loss of state between work sessions

## Rules
- create one file per session
- use `YYYY-MM-DD-session-##.md` naming
- every session must end with a carry-forward block
- link created assets and affected projects
- keep entries concise, factual, and reusable

## Standard Flow
1. define session objective
2. capture work completed
3. record decisions made
4. note created assets
5. define next highest-leverage step

## Required Files
- `session-template.md`
- dated session logs
